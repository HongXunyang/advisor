Feature anatomy

  • Domain: pure Python calculators in advisor/features/<feature>/domain/.
  • Controller: subclasses FeatureController, owns the calculator + view, forwards parameters.
  • View: PyQt tab(s) in advisor/features/<feature>/ui/, typically subclassing TabInterface.

Steps to add a feature

  1. Create domain code and export via __init__.py.
  2. Add a controller in features/<feature>/controllers/ extending FeatureController.
  3. Build the tab view in features/<feature>/ui/ (and ui/components/ if reusing widgets).
  4. Register the controller in AppController.features so the tab appears.
  5. Add an icon (optional) to resources/icons/ and set icon on the controller.

See docs/feature_development.md for a longer developer checklist.

Reusing shared pieces

  • Core geometry: advisor/domain/geometry.py
  • Lattice/Sample/Lab: advisor/domain/core/
  • Visualizers: advisor/ui/visualizers/
  • Tips/config: advisor/ui/tips.py and resources/config/tips.json

Styling and UX

  • QSS: resources/qss/styles.qss
  • Icons: resources/icons/
  • Add tooltips by extending resources/config/tips.json.
